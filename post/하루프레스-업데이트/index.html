하루프레스 업데이트(v0.9.2)

Monday, December 3 2012 3:38 PM
Tweet

하루프레스 v0.9.2 가 릴리즈 되었다.

  • 하루프레스 v0.9.2 회고 및 릴리즈 노트

내가 v0.9.1 을 설치한 후 이것 저것 해보다가 소뒷걸음질로 쥐잡듯 해결이 된 이슈(disqus 코멘트 개수)도 언급을 해주셨다. ㅜㅜ;; 이런 영광이... 이런 것이 동기부여가 좀 된다. ㅎㅎ

사실 설치한지도 얼마 되지 않았고, 버전 차이도 크지 않아서 업데이트 할 계획이 없었다. 더군다나 하루프레스를 내 저장소로 fork 한 후 clone 하여 설치한 것이라서 업데이트 할 엄두가 나지 않았다. git을 개인적으로 또 학습 위주로 사용했을 뿐이라서 자신이 없었다.

그런데 얼마전 아웃사이더님이 포스팅해 주신 글이 생각났다.

  • Github를 이용하는 전체 흐름 이해하기 #1
  • Github를 이용하는 전체 흐름 이해하기 #2

위 글을 보고 이참에 하루프레스를 업데이트 해보기로 했다. 왠지 자주 업데이트 할 일이 생길 것도 같고 ㅎㅎ

우선 전체 시나리오는 위의 글과 같이 하루프레스 소스 저장소에서 pull 한 뒤 내 저장소로 다시 push 하는 것이다.

그러기 위해서 우선 하루프레스 소스 저장소를 원격 저장소로 추가한 후 fetch와 merge를 수행했다.

$ git remote add haroo git@github.com:rhiokim/haroopress.git
$ git fetch haroo
$ git merge haroo/master

fetch와 merge 대신 git pull haroo로 처리해도 된다.

그리고 나서 나의 저장소로 push 해 주었다.

$ git push origin master

이슈 없이 업데이트가 끝난 줄 알았었는데 가만 보니 테마에 해당하는 부분은 업데이트가 되지 않았다.

하루프레스 테마 저장소가 별도 저장소로 되어 있어서 그런 것 같다. 같이 업데이트 되는 줄 알았는데... 테마 폴더로 이동 후 업데이트.

$ git pull origin master

이로써 하루프레스 v0.9.2 업데이트를 완료하고 이 글을 작성한다. :-)

아무래도 git에 대해서도 다시 봐야 할 것 같다.
